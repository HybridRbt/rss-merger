Black Forest 开源 FLUX.1 Kontex 模型

来源: 爱范儿

Black Forest Labs 本周正式开源图像编辑模型 FLUX.1 Kontext [dev]。 这是其专业版本 FLUX.1 Kontext [pro] 的开发者版，具备专有级图像编辑能力，参数规模达 120 亿，可在消费级硬件上运行。该模型已通过 HuggingFace 平台向研究与非商业用途用户开放下载，授权遵循 FLUX.1 非商业许可证。 FLUX.1 Kontext [dev] 聚焦图像编辑任务，支持精细化的局部与全局修改，能够在多样环境下保持角色一致性，并实现迭代式图像生成。 官方表示，该模型已在新发布的基准测试 KontextBench 上超越了包括 Bagel、HiDream-E1-Full，以及 Gemini-Flash Image 在内的多款开源与闭源竞品，性能获得第三方机构 Artificial Analysis 的独立验证。 为提升性能表现，FLUX.1 Kontext [dev] 针对英伟达最新的 Blackwell 架构进行了深度优化，并同步推出 BF16、FP8、FP4 等多种 TensorRT 变体，以满足不同场景下对速度、能效和图像质量的平衡需求。 Black Forest Labs 表示，该模型在推理速度和内存效率方面均有大幅提升。

多地机场：充电宝新规不影响携带锂电池

来源: 爱范儿

6 月 28 日起，民航局发布的「充电宝新规」正式生效：禁止旅客携带没有 3C 标识、3C 标识不清晰、被召回型号或批次的充电宝乘坐境内航班。 随着新规实施，不少网友也提出了「新规是否对锂电池的携带有影响」「国外生产的充电宝是否能携带」等问题。 报道指出，机场对相机锂电池的通用要求为无需 3C 认证，原装及备用裸电池容量一般需 ≤100Wh（或等同 ≤20000 mAh/5V），100-160Wh 的备用电池需航空公司批准，电池外壳须标识清晰，多数机场严禁托运。 目前，通过「充电宝 3C 标识」「3C 贴纸」等关键词搜索发现，大多数购物平台已经搜索不到相关商品，其中淘宝屏蔽了「3C 贴纸」搜索。据报道，部分平台已积极采取行动，持续识别并下架违规帖子，相关售卖帖数量在不断减少。 据悉，飞机客舱属于增压环境，在飞机起降过程中，客舱压力是变化的，会从起飞前的 1 个大气压降到巡航时的约 0.6 个大气压，降落时再慢慢增加到 1 个大气压。在这种环境下使用充电宝，容易导致发热、增加自燃的概率。

几十年来，Windows 允许安全软件等程序在内核层面运行。内核是操作系统的核心，对系统内存和硬件拥有最高权限，但也意味着一旦其驱动程序出现问题，极易导致系统崩溃，即「蓝屏」。大约一年前，安全公司 CrowdStrike 的错误更新曾引发全球系统瘫痪事件，约 850 万台 Windows 设备宕机。

微软还确认，将修改蓝屏的设计，改为黑色底色，类似安装 Windows 更新时的界面，但会显示错误代码和出错的驱动信息。

微软计划将杀毒软件移出 Windows 内核

据 The Verge 报道，微软正着手调整 Windows 架构，从而在将第三方杀毒软件和终端侦测与响应（EDR）程序移出 Windows 内核。

几十年来，Windows 允许安全软件等程序在内核层面运行。内核是操作系统的核心，对系统内存和硬件拥有最高权限，但也意味着一旦其驱动程序出现问题，极易导致系统崩溃，即「蓝屏」。大约一年前，安全公司 CrowdStrike 的错误更新曾引发全球系统瘫痪事件，约 850 万台 Windows 设备宕机。

微软表示，正与 CrowdStrike、Bitdefender、趋势科技等数十家安全厂商合作，共同设计新的安全平台。该项目目前已进入内部预览阶段，将根据安全厂商的反馈进行多轮迭代。除安全软件外，游戏反作弊引擎目前同样大量使用内核驱动，而微软也已开始与游戏开发者探讨减少内核依赖的方案，Riot Games 等厂商已表示对此持开放态度。

此外，为应对 CrowdStrike 一类事件导致的设备无法启动问题，微软计划在今年夏末的 Windows 更新中引入一项快速恢复功能。该功能可在设备无法启动时自动进入 Windows 恢复环境，通过网络连接向微软提供诊断信息，以实现快速修复。

微软还确认，将修改蓝屏的设计，改为黑色底色，类似安装 Windows 更新时的界面，但会显示错误代码和出错的驱动信息。

苹果进一步调整欧盟 App Store 规定

6 月 26 日，为了避免受到欧盟根据《数字市场法》的进一步处罚，苹果发布了新的欧洲专用 App Store 规则。

其中，最为显著的变化是新增 App Store 服务费分级机制。在欧洲，开发者可以选择只使用「第一级」服务，从而将商店服务费降至 5%。但这个层级仅提供最基本的功能，包括应用评论、隐私标签以及技术支持，不包括搜索推荐等引流功能，应用甚至不能自动更新。要继续使用现有的完整 App Store 功能，开发者需要选择「第二级」服务，支付 13% 的商店服务费。

苹果还开始统一欧洲专用规则下的收费方式，并准备用按比例收取的「核心技术佣金」（每次交易的 5%）取代之前按设备数定额收取的「核心技术费」（每次安装 0.5 欧元）。根据旧规则，苹果按照三种不同方式对欧洲开发者收费，具体取决于开发者签订的协议种类、是否使用第三方支付服务，以及交易属于应用内购还是外部渠道等。而根据新规则，苹果总是收取初始获客费、商店服务费和核心技术佣金。基于核心技术费的计算方式将暂时保留，直到 2026 年完全过渡到基于核心技术佣金的计算方式。

此前，欧盟裁定苹果 App Store 仍在不当限制开发者引导用户至外部渠道消费，违反了《数字市场法》，并处以 5 亿欧元罚款。欧盟还发布了一项初步调查，认为苹果先前规则不是「必要且适度」的。尽管修改了规则，苹果仍表示计划对上述处罚提起上诉。欧盟委员会则表示将评估苹果提出的新规则，然后决定是否施加额外罚款。

几十年来，Windows 允许安全软件等程序在内核层面运行。内核是操作系统的核心，对系统内存和硬件拥有最高权限，但也意味着一旦其驱动程序出现问题，极易导致系统崩溃，即「蓝屏」。大约一年前，安全公司 CrowdStrike 的错误更新曾引发全球系统瘫痪事件，约 850 万台 Windows 设备宕机。

微软表示，正与 CrowdStrike、Bitdefender、趋势科技等数十家安全厂商合作，共同设计新的安全平台。该项目目前已进入内部预览阶段，将根据安全厂商的反馈进行多轮迭代。除安全软件外，游戏反作弊引擎目前同样大量使用内核驱动，而微软也已开始与游戏开发者探讨减少内核依赖的方案，Riot Games 等厂商已表示对此持开放态度。

此外，为应对 CrowdStrike 一类事件导致的设备无法启动问题，微软计划在今年夏末的 Windows 更新中引入一项快速恢复功能。该功能可在设备无法启动时自动进入 Windows 恢复环境，通过网络连接向微软提供诊断信息，以实现快速修复。

微软还确认，将修改蓝屏的设计，改为黑色底色，类似安装 Windows 更新时的界面，但会显示错误代码和出错的驱动信息。

苹果进一步调整欧盟 App Store 规定

6 月 26 日，为了避免受到欧盟根据《数字市场法》的进一步处罚，苹果发布了新的欧洲专用 App Store 规则。

其中，最为显著的变化是新增 App Store 服务费分级机制。在欧洲，开发者可以选择只使用「第一级」服务，从而将商店服务费降至 5%。但这个层级仅提供最基本的功能，包括应用评论、隐私标签以及技术支持，不包括搜索推荐等引流功能，应用甚至不能自动更新。要继续使用现有的完整 App Store 功能，开发者需要选择「第二级」服务，支付 13% 的商店服务费。

苹果还开始统一欧洲专用规则下的收费方式，并准备用按比例收取的「核心技术佣金」（每次交易的 5%）取代之前按设备数定额收取的「核心技术费」（每次安装 0.5 欧元）。根据旧规则，苹果按照三种不同方式对欧洲开发者收费，具体取决于开发者签订的协议种类、是否使用第三方支付服务，以及交易属于应用内购还是外部渠道等。而根据新规则，苹果总是收取初始获客费、商店服务费和核心技术佣金。基于核心技术费的计算方式将暂时保留，直到 2026 年完全过渡到基于核心技术佣金的计算方式。

此前，欧盟裁定苹果 App Store 仍在不当限制开发者引导用户至外部渠道消费，违反了《数字市场法》，并处以 5 亿欧元罚款。欧盟还发布了一项初步调查，认为苹果先前规则不是「必要且适度」的。尽管修改了规则，苹果仍表示计划对上述处罚提起上诉。欧盟委员会则表示将评估苹果提出的新规则，然后决定是否施加额外罚款。

据 The Verge 报道，微软正着手调整 Windows 架构，从而在将第三方杀毒软件和终端侦测与响应（EDR）程序移出 Windows 内核。

几十年来，Windows 允许安全软件等程序在内核层面运行。内核是操作系统的核心，对系统内存和硬件拥有最高权限，但也意味着一旦其驱动程序出现问题，极易导致系统崩溃，即「蓝屏」。大约一年前，安全公司 CrowdStrike 的错误更新曾引发全球系统瘫痪事件，约 850 万台 Windows 设备宕机。

微软表示，正与 CrowdStrike、Bitdefender、趋势科技等数十家安全厂商合作，共同设计新的安全平台。该项目目前已进入内部预览阶段，将根据安全厂商的反馈进行多轮迭代。除安全软件外，游戏反作弊引擎目前同样大量使用内核驱动，而微软也已开始与游戏开发者探讨减少内核依赖的方案，Riot Games 等厂商已表示对此持开放态度。

此外，为应对 CrowdStrike 一类事件导致的设备无法启动问题，微软计划在今年夏末的 Windows 更新中引入一项快速恢复功能。该功能可在设备无法启动时自动进入 Windows 恢复环境，通过网络连接向微软提供诊断信息，以实现快速修复。

微软还确认，将修改蓝屏的设计，改为黑色底色，类似安装 Windows 更新时的界面，但会显示错误代码和出错的驱动信息。

其中，最为显著的变化是新增 App Store 服务费分级机制。在欧洲，开发者可以选择只使用「第一级」服务，从而将商店服务费降至 5%。但这个层级仅提供最基本的功能，包括应用评论、隐私标签以及技术支持，不包括搜索推荐等引流功能，应用甚至不能自动更新。要继续使用现有的完整 App Store 功能，开发者需要选择「第二级」服务，支付 13% 的商店服务费。

苹果还开始统一欧洲专用规则下的收费方式，并准备用按比例收取的「核心技术佣金」（每次交易的 5%）取代之前按设备数定额收取的「核心技术费」（每次安装 0.5 欧元）。根据旧规则，苹果按照三种不同方式对欧洲开发者收费，具体取决于开发者签订的协议种类、是否使用第三方支付服务，以及交易属于应用内购还是外部渠道等。而根据新规则，苹果总是收取初始获客费、商店服务费和核心技术佣金。基于核心技术费的计算方式将暂时保留，直到 2026 年完全过渡到基于核心技术佣金的计算方式。

微软计划将杀毒软件移出 Windows 内核

据 The Verge 报道，微软正着手调整 Windows 架构，从而在将第三方杀毒软件和终端侦测与响应（EDR）程序移出 Windows 内核。

几十年来，Windows 允许安全软件等程序在内核层面运行。内核是操作系统的核心，对系统内存和硬件拥有最高权限，但也意味着一旦其驱动程序出现问题，极易导致系统崩溃，即「蓝屏」。大约一年前，安全公司 CrowdStrike 的错误更新曾引发全球系统瘫痪事件，约 850 万台 Windows 设备宕机。

微软表示，正与 CrowdStrike、Bitdefender、趋势科技等数十家安全厂商合作，共同设计新的安全平台。该项目目前已进入内部预览阶段，将根据安全厂商的反馈进行多轮迭代。除安全软件外，游戏反作弊引擎目前同样大量使用内核驱动，而微软也已开始与游戏开发者探讨减少内核依赖的方案，Riot Games 等厂商已表示对此持开放态度。

此外，为应对 CrowdStrike 一类事件导致的设备无法启动问题，微软计划在今年夏末的 Windows 更新中引入一项快速恢复功能。该功能可在设备无法启动时自动进入 Windows 恢复环境，通过网络连接向微软提供诊断信息，以实现快速修复。

微软还确认，将修改蓝屏的设计，改为黑色底色，类似安装 Windows 更新时的界面，但会显示错误代码和出错的驱动信息。

苹果进一步调整欧盟 App Store 规定

6 月 26 日，为了避免受到欧盟根据《数字市场法》的进一步处罚，苹果发布了新的欧洲专用 App Store 规则。

其中，最为显著的变化是新增 App Store 服务费分级机制。在欧洲，开发者可以选择只使用「第一级」服务，从而将商店服务费降至 5%。但这个层级仅提供最基本的功能，包括应用评论、隐私标签以及技术支持，不包括搜索推荐等引流功能，应用甚至不能自动更新。要继续使用现有的完整 App Store 功能，开发者需要选择「第二级」服务，支付 13% 的商店服务费。

苹果还开始统一欧洲专用规则下的收费方式，并准备用按比例收取的「核心技术佣金」（每次交易的 5%）取代之前按设备数定额收取的「核心技术费」（每次安装 0.5 欧元）。根据旧规则，苹果按照三种不同方式对欧洲开发者收费，具体取决于开发者签订的协议种类、是否使用第三方支付服务，以及交易属于应用内购还是外部渠道等。而根据新规则，苹果总是收取初始获客费、商店服务费和核心技术佣金。基于核心技术费的计算方式将暂时保留，直到 2026 年完全过渡到基于核心技术佣金的计算方式。
